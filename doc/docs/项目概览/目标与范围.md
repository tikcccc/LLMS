## 1. 项目目标

### 1.1 业务目标

| 目标 | 描述 | 衡量标准 |
|------|------|----------|
| 统一数据视图 | 将地块规划数据、官方图层、内部规划图层在一个 Web 端统一呈现并可对比 | 所有相关图层可在同一地图界面查看和切换 |
| 任务全流程闭环 | 从立项、派工、执行到验收，流程清晰可追踪 | 任务状态实时更新，进度可追溯 |
| 降低协调成本 | 通过地图可视化和移动端工具，缩短现场协作链路 | 现场人员可独立查询地块信息和任务 |
| 风险预警 | 实时显示任务状态，对逾期和困难个案进行预警 | 逾期任务自动标红，困难个案可标记 |

### 1.2 用户目标

**管理者/规划人员**：
- ✅ 在地图上快速创建和维护 Work Lot
- ✅ 为地块分配任务并追踪进度
- ✅ 查看全局状态和逾期风险
- ✅ 按区域查询和导出报表

**现场人员**：
- ✅ 在手机上查看分配的任务
- ✅ 到现场后通过 GPS 核验位置
- ✅ 查看地块信息和任务要求
- ✅ 更新任务状态

### 1.3 核心价值（Demo 必须传达的 4 个核心价值）

1. **画图**：创建地块（Land Lot）和运营范围（Work Lot）
2. **改图**：编辑边界和属性
3. **分层管理**：法律地界（Land Lot）vs 营运范围（Work Lot）
4. **任务指派**：Work Lot 关联任务并追踪执行

## 3. 范围边界说明

### 3.1 Land Lot vs Work Lot

| 维度 | Land Lot（地段） | Work Lot（作业地块） |
|------|------------------|---------------------|
| 定义 | 地政总署定义的法律地界 | 用户绘制的实际占用范围 |
| 数据来源 | 政府记录（Soft Copy Plans） | 用户手动绘制 |
| 可编辑性 | 可编辑（管理员权限） | 可编辑（Site Officer 权限） |
| 用途 | 参考图层（Reference Layer） | 操作图层（Active Layer） |
| 关系 | 多对多（一个 Work Lot 可跨多个 Land Lot） | 一对一（一个 Work Lot 代表一个 Operator） |

### 3.2 数据精度说明

**初期（Week 1-8）**：
- 使用 CEDD 提供的 Soft Copy Plans（PDF/图片）
- 人工描图输入系统
- 边界为示意性质（Indicative），非测量级精度
- 系统需明确标注数据来源和精度限制

**后期（Phase 2+）**：
- 测量队提供精确坐标
- 支持导入 GIS/CAD 格式数据
- 逐步替换示意图为精确边界

## 7. 风险与对策

| 风险 | 影响 | 概率 | 对策 |
|------|------|------|------|
| CSDI 可用性/限流 | 地图无法加载 | 中 | 缓存底图、失败降级提示、重试机制 |
| 数据精度不足 | 边界判断误差 | 高 | 明确标注示意性质、支持后期更新 |
| 服务器访问延迟 | 影响开发进度 | 中 | 提前协调、准备本地开发环境 |
| 大量地块渲染性能下降 | 地图卡顿 | 中 | 按视窗加载、矢量切片、简化几何 |
| 移动端分发成本 | 更新困难 | 低 | 企业签名/MDM 分发、版本检查 |

## 8. 变更管理

### 8.1 范围变更流程

1. 提出变更请求（书面）
2. 评估影响（时间、成本、技术）
3. 双方确认（签字）
4. 更新文档和计划

### 8.2 优先级调整原则

- **P0（必须）**：影响核心价值的功能（画图、改图、分层管理、任务指派）
- **P1（重要）**：影响用户体验的功能（搜索、导出、GPS 核验）
- **P2（可选）**：锦上添花的功能（高级报表、附件上传）

变更时优先保证 P0，可协商调整 P1/P2。
