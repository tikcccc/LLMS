# GIS 方案

## 1. 目标与范围
- 统一底图与坐标口径，支持地块绘制、查询、核验与展示。
- 覆盖 Web 管理端与移动端的地图能力与 GIS 规则。

## 2. 坐标系口径
- 统一使用 HK80（EPSG:2326）作为系统坐标口径。
- 外部数据导入时需转换到 HK80，并保留坐标系标识。
- 前端绘制与后端存储均以 HK80 坐标传递。

## 3. 底图与标注接入
- 底图：CSDI Topographic Map API（HK80，zoom 10-20）。
- 标注：CSDI Map Label API（HK80，zoom 8-20，语言 tc/en/sc）。
- 版权：地图界面必须展示 CSDI/LandsD 署名与版权信息。

## 4. 业务图层
- LandLot：法律地界参考层（只读、蓝框）。
- WorkLot：运营范围编辑层（彩色面、可编辑）。
- Task：任务点/面标记层（与 WorkLot 关联）。

## 5. 空间计算与规则
- WorkLot 与 LandLot 关系以相交/包含为准，落库关联。
- 绘制合法性检查：闭合、自交、最小面积阈值。
- 面积/长度计算基于 HK80 坐标，展示与导出一致。

## 6. 数据采集与更新
- Web 端负责绘制与维护 WorkLot、LandLot 关联。
- 移动端以只读/核验为主，必要时支持现场定位。
- Soft Copy Plans 或外部文件导入前需完成坐标转换与字段映射。

## 7. 性能与稳定性
- 地图请求节流与失败降级提示，避免短时间大量请求 CSDI API。
- 空间查询使用空间索引，保证地图筛选与列表联动性能。
